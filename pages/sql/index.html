1. Выборка данных

SELECT: используется для выбора данных из базы данных

  • SELECT * FROM имя_таблицы;

DISTINCT: отфильтровывает повторяющиеся значения и возвращает строки определенного столбца

  • SELECT DISTINCT имя_столбца;

WHERE: используется для фильтрации записей / строк

  • SELECT столбец1, столбец2 FROM имя_таблицы WHERE условие;
  • SELECT * FROM имя_таблицы WHERE условие1 AND условие2;
  • SELECT * FROM имя_таблицы WHERE условие1 OR условие2;
  • SELECT * FROM имя_таблицы WHERE NOT условие;
  • SELECT * FROM имя_таблицы WHERE условие1 AND (условие2 OR условие3);
  • SELECT * FROM имя_таблицы WHERE EXISTS (SELECT имя_столбца FROM имя_таблицы WHERE условие);

ORDER BY: используется для сортировки набора результатов в порядке возрастания или убывания

  • SELECT * FROM имя_таблицы ORDER BY столбец;
  • SELECT * FROM имя_таблицы ORDER BY столбец DESC;
  • SELECT * FROM имя_таблицы ORDER BY столбец1 ASC, столбец2 DESC;

SELECT TOP: используется для указания количества записей, возвращаемых из верхней части таблицы

  • SELECT TOP number имена_столбцов FROM имя_таблицы WHERE условие;
  • SELECT TOP percent имена_столбцов FROM имя_таблицы WHERE условие;
  • Не все системы баз данных поддерживают SELECT TOP. Эквивалентом MySQL является LIMIT
  • SELECT имена_столбцов FROM имя_таблицы LIMIT смещение, кол-во;

LIKE: оператор, используемый в предложении WHERE для поиска определенного шаблона в столбце

  • % (Знак процента) является символом подстановки, который представляет ноль, один или несколько символов
  • _ (подчеркивание) - это символ подстановки, который представляет один символ
  • SELECT имена_столбцов FROM имя_таблицы WHERE имя_столбца LIKE шаблон;
  • LIKE ‘a%’ (Найти любые значения, начинающиеся с «а»)
  • LIKE ‘%a’ (найти любые значения, которые заканчиваются на «а»)
  • LIKE ‘%or%’ (найти любые значения, которые имеют «or» в любой позиции)
  • LIKE ‘_r%’ (найти любые значения, которые имеют «r» во второй позиции)
  • LIKE ‘a_%_%’ (найти любые значения, которые начинаются с «а» и имеют длину не менее 3 символов)
  • LIKE ‘[a-c]%’ (найти любые значения, начинающиеся с «a», «b» или «c»)

IN: оператор, который позволяет указать несколько значений в предложении WHERE

  • По сути, оператор IN является сокращением для нескольких условий OR
  • SELECT имена_столбцов FROM имя_таблицы WHERE имя_столбца IN (значение1, значение2, …);
  • SELECT имена_столбцов FROM имя_таблицы WHERE имя_столбца IN (SELECT STATEMENT);

BETWEEN: оператор выбирающий значения в заданном диапазоне включительно

  • SELECT имена_столбцов FROM имя_таблицы WHERE имя_столбца BETWEEN значение1 AND значение2;
  • SELECT * FROM Продукты WHERE (имя_столбца1 BETWEEN значение1 AND значение2) AND NOT имя_столбца2 IN (значение3, значение4);
  • SELECT * FROM Продукты WHERE имя_столбца BETWEEN #01/07/1999# AND #03/12/1999#;

NULL: значения в поле без значения

  • SELECT * FROM имя_таблицы WHERE имя_столбца IS NULL;
  • SELECT * FROM имя_таблицы WHERE имя_столбца IS NOT NULL;

AS: псевдонимы, которые используются для назначения временного имени таблице или столбцу

  • SELECT имя_столбца AS имя_псевдонима FROM имя_таблицы;
  • SELECT имя_столбца FROM имя_таблицы AS имя_псевдонима;
  • SELECT имя_столбца AS имя_псевдонима1, имя_столбца2 AS имя_псевдонима2;
  • SELECT имя_столбца1, имя_столбца2 + ‘, ‘ + имя_столбца3 AS имя_псевдонима;

UNION: используется для объединения результирующего набора из двух или более операторов SELECT

  • Каждый оператор SELECT в UNION должен иметь одинаковое количество столбцов
  • Столбцы должны иметь одинаковые типы данных
  • Столбцы в каждом операторе SELECT также должны быть в том же порядке
  • SELECT имена_столбцов FROM таблица1 UNION SELECT имя_столбца FROM таблица2;
  • UNION оператор выбирает только отдельные значения, UNION ALL позволит дубликаты

INTERSECT: оператор, который используется для возврата записей, которые имеют два общих оператора SELECT

  • Обычно используется так же, как и UNION
  • SELECT имена_столбцов FROM таблица1 INTERSECT SELECT имя_столбца FROM таблица2;

EXCEPT: используется для возврата всех записей в первом операторе SELECT, которые не найдены во втором операторе SELECT

  • Обычно используется так же, как и UNION
  • SELECT имена_столбцов FROM таблица1 EXCEPT SELECT имя_столбца FROM таблица2;

ANY|ALL: оператор, используемый для проверки условий подзапроса, используемых в предложениях WHERE или HAVING

  • Оператор ANY возвращает true, если какие-либо значения подзапроса удовлетворяют условию
  • Оператор ALL возвращает true, если все значения подзапроса удовлетворяют условию
  • SELECT имена_столбцов FROM таблица1 WHERE имя_столбца оператор (ANY|ALL) (SELECT имя_столбца FROM имя_таблицы WHERE условие);

GROUP BY: оператор, оператор, часто используемый с агрегатными функциями (COUNT, MAX, MIN, SUM, AVG) для группировки результирующего набора по одному или нескольким столбцам

  • SELECT имя_столбца1, COUNT(имя_столбца2) FROM имя_таблицы WHERE условие GROUP BY имя_столбца1 ORDER BY COUNT(имя_столбца2) DESC;

HAVING: это предложение было добавлено в SQL, поскольку ключевое слово WHERE нельзя использовать с агрегатными функциями

  • SELECT COUNT(имя_столбца1), имя_столбца2 FROM таблица GROUP BY имя_столбца2 HAVING COUNT(имя_столбца1) > 5;

WITH: часто используется для извлечения иерархических данных или повторного использования временного набора результатов в запросе несколько раз. Также называется «общим выражением таблицы» (овт)

  • WITH RECURSIVE овт AS (
      SELECT о0.* FROM категории AS о0 WHERE id = 1 # Отправная точка
      UNION ALL
      SELECT c1.* FROM категории AS о1 JOIN овт ON о1.идентификатор_категории_id = овт.id
    )
    SELECT *
    FROM овт
